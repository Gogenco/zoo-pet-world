# Stage 4.4B — Coloring Image + Pairs Map Architecture Fix

## База

`ZooPetWorld_stage4_4A_coloring_portrait_refactor.zip`

## Что исправлено

### 1. Раскраска: картинка не появлялась

Причина: после Stage 4.4A был удалён старый CSS-патч, внутри которого был важный стиль для `#coloringCanvasViewport`.  
Этот viewport создаётся модулем zoom/pinch и содержит canvas-ы раскраски. Без его размеров/позиционирования картинка могла не отображаться корректно.

Исправлено:

- восстановлен нужный CSS для `#coloringCanvasViewport`;
- при этом зелёные scroll-кнопки не возвращались;
- старый вызов `buildColoringScrollHandles()` удалён;
- pinch/tap-to-fill сохранены.

### 2. Карта уровней перенесена в правильный сценарий

Теперь карта уровней относится только к игре **Найди пары**.

Правильный путь:

`Найди пары → выбор режима → карта уровней`

Исправлено:

- на главном экране удалён прямой вход в `mapScreen`;
- из экрана выбора режима удалена кнопка “Карта” до выбора режима;
- карта открывается только после выбора:
  - спокойный режим;
  - режим с таймером;
- из игры и victory modal кнопка “Карта уровней” сохранена, потому что это уже контекст игры “Найди пары”.

### 3. Главный экран: новая будущая игра

На месте старой кнопки “Карта уровней” теперь карточка:

`Найди отличия`

Пока это planned/soon игра:

- открывает маленький teaser popup;
- не запускает новую игру;
- не ведёт на карту уровней.

### 4. Тексты карты

Заменено:

- `Карта зоопарка` → `Карта уровней`
- subtitle → `Уровни игры «Найди пары» — выбери режим и проходи по порядку`
- закрытый уровень → `Закрыто. Пройди предыдущий уровень.`
- описание уровня больше не строится вокруг “вольеров зоопарка”.
