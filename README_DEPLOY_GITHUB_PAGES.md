# Zoo Pet World — GitHub Pages Deployment

## Что внутри

```
zoo-pet-world-pages/
├── index.html        (599 KB — главный файл игры)
├── assets/
│   ├── inline/       (54 файла: спрайты, фото питомцев, фоны)
│   └── js/           (модули раскраски, экономики, фич)
└── README_DEPLOY_GITHUB_PAGES.md
```

## Деплой на GitHub Pages (вручную, ~5 минут)

### Шаг 1 — Создать репозиторий
1. Зайди на **github.com** → войди в аккаунт
2. Нажми **"+"** → **"New repository"**
3. Имя: `zoo-pet-world` (или любое другое)
4. Тип: **Public** ← обязательно для бесплатного Pages
5. Галочки НЕ ставить (без README, без .gitignore)
6. Нажми **"Create repository"**

### Шаг 2 — Загрузить файлы
1. На странице репозитория нажми **"uploading an existing file"**
2. Перетащи ВСЕ файлы из этой папки:
   - `index.html`
   - папку `assets/` целиком
3. Нажми **"Commit changes"**

> ⚠️ Важно: нужно загрузить папку `assets/` со всей структурой внутри,
> а не только файлы из неё. GitHub позволяет drag-and-drop папок.

### Шаг 3 — Включить GitHub Pages
1. В репозитории → **Settings** (вкладка сверху)
2. Слева в меню → **Pages**
3. Branch: выбери **main**, папка **/ (root)**
4. Нажми **Save**

### Шаг 4 — Открыть игру
Через 1-2 минуты игра будет доступна по адресу:
```
https://ТВО_ЛОГИН.github.io/zoo-pet-world/
```

## Почему GitHub Pages лучше чем file://

| | file:// (сейчас) | GitHub Pages (HTTPS) |
|---|---|---|
| Кэш браузера | ❌ нет | ✅ все ресурсы кэшируются |
| Повторное открытие | ~3с каждый раз | <0.5с из кэша |
| Canvas без ошибок | ⚠️ ограничения | ✅ полный доступ |
| Safari на iOS | ⚠️ file:// проблемы | ✅ работает идеально |
| Добавить на домашний экран | ❌ | ✅ как приложение |

## Добавить на домашний экран iOS (после деплоя)

1. Открой URL игры в **Safari**
2. Нажми кнопку **«Поделиться»** (квадрат со стрелкой)
3. Выбери **«На экран "Домой"»**
4. Игра запускается как приложение без адресной строки
